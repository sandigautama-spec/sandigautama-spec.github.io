ModernVet ETL Sync API

Push records from AWS S3 (or local files) into Odoo 18. Processing is asynchronous — trigger once, poll for result.

Odoo 18.0 Async via Cron API Key Auth AWS S3 + Local
How to Use
Everything you need to get started with the ETL sync module

→
📡 POST /api/v1/etl/notifications

📋 Log created (queued)

1 Authentication

Pass your API key in either header:

  • X-API-Key: <key>
  • Authorization: Bearer <key>

Keys are managed in Odoo under Settings → ETL API Keys (model: etl.api.key).

2 Trigger a Sync

POST a JSON body to /api/v1/etl/notifications:

  • data_type — what to sync
  • s3_location — S3 bucket/prefix or local path
  • local: true — read from filesystem

Returns {"status":"queued","log_id":42} immediately.

3 Poll for Status

GET /api/v1/etl/logs/{log_id} to check progress. States: queued → running → done / failed.

  • done — check created / updated / skipped / errors counters
  • failed — read error field for exception details

4 S3 Location Resolution

  1. Try s3_location as a direct object key
  2. If not found, list all objects under that prefix
  3. Download and process all matching files together

Multiple files are concatenated before parsing.

5 File Format

Files must be JSON array or JSONL (one object per line). Every record must have id_from_ext as the upsert key.

  • Encoding: UTF-8
  • Extension: .json or .jsonl
  • Max size: limited by available memory

6 Idempotency

All syncs are safe to re-run. Records are matched by id_from_ext:

  • Product / Sales / Invoice — upsert (create or update)
  • Loyalty History — create-only; existing id_from_ext is skipped
  • Posted/cancelled invoices — skipped on update attempt

7 AWS S3 Credentials

Configure in Odoo Settings → Technical → System Parameters:

  • pet_sync_etl.aws_region — default: ap-southeast-3
  • pet_sync_etl.aws_access_key_id
  • pet_sync_etl.aws_secret_access_key

Leave key/secret empty to use IAM role credentials.

8 Loyalty Journal Config

Required for loyalty journal entries. Configure per company in Settings → Accounting → Loyalty Configuration:

  • Invoice journal, payable account, revenue deduction account
  • Non-invoice journal, payable account, revenue deduction account

9 Stuck Jobs & Auto-Reset

If a job stays in running state for more than 2 hours, the next cron run auto-resets it to failed.

Cron runs every ~1 minute and processes one queued log at a time to prevent race conditions.

Invoice
data_type: "invoice" or "invoices"  ·  Target model: account.move  ·  Upsert by id_from_ext
ℹ️
Auto-post flow: When state=posted, the ETL will:
  1. Add loyalty redemption line if point_redeem_idr > 0
  2. Add global discount line if global_discount < 0
  3. Post the invoice
  4. Create loyalty card history + journal entries
  5. Register payments from payment_list (actual amount = paid_amount − deduction lines)

Loyalty History
data_type: "loyalty_history"  ·  Target models: loyalty.card + loyalty.history  ·  Create-only by id_from_ext
⚠️
Create-only: Records where id_from_ext already exists are silently skipped — no update is performed. This prevents duplicate journal entries on re-sync.
⚙️
Journal entry logic: Requires company's non-invoice loyalty journal to be configured in Settings → Accounting → Loyalty Configuration.
  • issued > 0 → Earn JE: Debit Revenue Deduction account, Credit Payable account
  • used > 0 → Redeem JE: Debit Payable account, Credit Revenue Deduction account

Body FieldTypeRequiredDescription
data_typeenumRequired"product", "products", "sales", "sale", "invoice", "invoices", "loyalty_history"
